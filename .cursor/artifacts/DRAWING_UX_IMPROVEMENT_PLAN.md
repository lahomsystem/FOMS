### 1.2 프로세스 문서 대비 현재 구현 GAP 분석

| # | 프로세스 문서 요구사항 | 현재 구현 상태 | GAP |

| 11 | 최종적으로 최종 도면만 첨부파일 '도면'에 존재 | 교체 시에만 이전 삭제 | **추가 모드에서 누적됨** |

### 1.3 UI/UX 상세 문제점

#### A. 도면작업실 대시보드 (`erp_drawing_workbench_dashboard.html`)

| 문제 | 심각도 | 설명 |
|------|-------|------|
| 테이블 정렬 불가 | 중 | SLA, 상태, 미확인 등 컬럼 클릭 정렬 없음 |
| 페이지네이션 없음 | 중 | 대량 데이터 시 성능/스크롤 문제 |
| 일괄 선택/작업 없음 | **높** | 여러 주문 체크 후 일괄 담당자 할당 불가 |
| 담당자 할당 버튼 없음 | **높** | 대시보드에서 바로 할당 불가, 상세 이동 필요 |
| 필터 상태 저장 없음 | 낮 | 페이지 이동 후 복귀 시 필터 초기화 |
| 요약 통계 없음 | 중 | 상태별 건수, 미확인 합계 등 한눈에 파악 불가 |

#### B. 도면작업실 실행판 (`erp_drawing_workbench_detail.html`)

| 문제 | 심각도 | 설명 |
|------|-------|------|
| 제품 정보 열람 불가 | **높** | ERP Beta 제품 정보(아이템, 치수 등) 직접 노출 없음 |
| 실측 사진 열람 불가 | **높** | 실측 사진이 실행판 내에서 바로 보이지 않음 |
| 담당자 변경 UI 없음 | **높** | 도면 담당자 재할당/추가 불가 |
| 파일 업로드 진행률 없음 | 중 | 대용량 도면 업로드 시 피드백 없음 |
| 드래그앤드롭 미지원 | 중 | 파일 선택만 가능 |
| 교체 모드 UI 직관성 부족 | 중 | 드롭다운만 제공, 카드형 선택 없음 |
| 수정 요청 참고 사진 미리보기 없음 | 중 | 첨부 선택 후 확인 불가 |
| 모바일 sticky 동작 이상 | 낮 | 992px 이하에서 사이드바 위치 문제 |

#### C. ERP 대시보드 도면 창구 (`erp_dashboard.html`)

| 문제 | 심각도 | 설명 |
|------|-------|------|
| 담당자 할당 모달 - 현재 할당자 체크 미표시 | **높** | 코드 3389-3396줄 주석 처리됨 |
| 도면 카드형 선택 UI ↔ 작업실 드롭다운 불일치 | 중 | UX 일관성 부재 |
| 도면 창구가 테이블 셀 내부 | 중 | 반응형 제약, 모바일 UX 저하 |

#### D. 백엔드 API (`apps/erp_beta.py`)

| 문제 | 심각도 | 설명 |
|------|-------|------|
| 일괄 담당자 할당 API 없음 | **높** | 여러 주문 동시 할당 불가 |
| 다중 도면 교체 API 없음 | 중 | 1개씩만 교체 가능 |
| 파일 삭제 실패 시 롤백 없음 | 중 | 2802-2803줄, 삭제 실패해도 진행 |
| `CONFIRMED` 상태 수정 요청 허용 여부 불명확 | 낮 | 3191줄 주석만 있음 |

## 2부: 개선사항 도출 (우선순위별)

### P0 (즉시 - 실무 프로세스 차단 해소)

#### [P0-1] 도면작업실에서 담당자 직접 할당
- **현재**: ERP 대시보드로 이동해야만 할당 가능
- **개선**: 도면작업실 대시보드 + 실행판 양쪽에 할당 버튼/모달 추가
- **근거**: 프로세스 문서 "도면 할당 버튼을 '도면 작업실'로 이동"

#### [P0-2] 일괄 담당자 할당
- **현재**: 주문 1건씩 개별 할당만 가능
- **개선**: 대시보드에서 체크박스로 여러 주문 선택 → 일괄 할당
- **근거**: 프로세스 문서 "각 주문 1:1 개별 할당 or 여러 주문건 선택 후 할당"

#### [P0-3] 제품 정보 + 실측 사진 인라인 표시
- **현재**: 별도 페이지(ERP Beta edit_order)로 이동해야 확인
- **개선**: 실행판 사이드바에 "참고 자료" 섹션 추가 (제품 정보 텍스트 + 실측 사진 갤러리)
- **근거**: 프로세스 문서 "제품 정보 및 실측 사진 자료를 주문 건 마다 확인"

#### [P0-4] 담당자 할당 모달 - 현재 할당자 체크 표시
- **현재**: 모달 열면 항상 전부 미체크
- **개선**: structured_data에서 현재 할당자 정보를 읽어 체크 표시

### P1 (이번 스프린트 - UX 핵심 개선)

#### [P1-1] 다중 도면 선택 수정 지원
- **현재**: 수정 요청/교체 시 1개 도면만 선택 가능
- **개선**: 체크박스/카드형 다중 선택 → N개 동시 교체/수정 요청
- **근거**: 프로세스 문서 "여러 도면 중 N개의 도면 수정 시 수정된 도면만 이전 파일 삭제"

#### [P1-2] 수정 도면 전체 교체 시 자동 삭제
- **현재**: replace_target_key 지정 시에만 이전 파일 삭제
- **개선**: "전체 교체" 옵션 추가 → 모든 기존 도면 자동 삭제 후 새 파일로 대체
- **근거**: 프로세스 문서 "최종적으로 최종 도면만 첨부파일 '도면'에 첨부 되게"

#### [P1-3] 파일 업로드 UX 개선
- 업로드 진행률 바 표시
- 드래그앤드롭 지원
- 선택한 파일 미리보기 (이미지 썸네일)
- 파일 크기/타입 검증 피드백

#### [P1-4] 도면 선택 UI 통일 (카드형)
- **현재**: ERP 대시보드는 카드형, 작업실은 드롭다운
- **개선**: 양쪽 모두 카드형 UI 통일 (썸네일 + 번호 + 파일명)

#### [P1-5] 대시보드 요약 통계 카드
- 상태별 건수 (작업중 / 확정대기 / 수정요청 / 완료)
- SLA 지연 건수
- 미확인 요청 합계

### P2 (차기 스프린트 - 생산성 향상)

#### [P2-1] 대시보드 테이블 정렬/페이지네이션
- 컬럼 헤더 클릭 정렬
- 서버사이드 페이지네이션 (25건/페이지)

#### [P2-2] 필터 상태 URL 저장/복원
- 필터 변경 시 URL 파라미터 업데이트
- 뒤로가기/북마크 시 필터 유지

#### [P2-3] 도면 관련 JS 파일 분리
- 인라인 스크립트 → `static/js/erp/drawing-workbench.js` 분리
- 코드 재사용성/유지보수성 향상

#### [P2-4] 모바일 반응형 최적화
- 대시보드 테이블 → 카드형 레이아웃 전환
- 실행판 sticky 사이드바 → 하단 고정 바
- 터치 친화적 버튼 크기 확대
