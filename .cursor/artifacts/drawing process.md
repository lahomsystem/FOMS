#도면 작업 프로세스
!!!도면작업자 들은 ‘도면작업실’을 메인으로 본다

1. 제작할 도면 확인 : workflow가 ‘도면’으로 넘어오면 > 도면 작업 할 주문건들이 도면작업실 대시보드에 표시

1. 영업/실측 담당이 현장 정보를 바탕으로 입력 한 > Erpbeta >’제품 정보’ 및 ‘실측 사진’ 자료를 주문 건 마다 확인

2. 확인 후 도면팀에서 자율적으로 ‘담당자’를 할당 한다 
**도면 할당 버튼 및 프로세스를 현재 ‘대시보드 작업 큐 도면 창구 요약’에서 ‘도면 작업실’로 이동

3. 각 주문 1:1 개별 할당 or 여러 주문건 선택 후 할당

4. Erpbeta >’제품 정보’ 및 ‘실측 사진’ 자료를 바탕으로 도면 제작

5. 제작 된 도면을 발송(각 할당 된 담당자(A)에게 자동 발송 구현 돼 있음)

6. A는 확인 후 
6.1 도면 이상없으면 컨펌 -> workflow 상 고객 컨펌으로 이동
6.2 도면 수정 필요 하면 -> 수정 요청(뭘 수정 해야 되는지 내용과, 참고 사진 업로드 프로세스 필요)

7. 수정요청을 대시보드에서 확인(내용, 참고 사진 바탕으로)

8. 요청된 사항을 바탕으로 수정 도면 발송|
8.1 컨펌 시 또는 수정 시 6의 프로세스 반복(티키타카)

*수정 도면 발송 시 첨부파일>’도면’ 의 이전 파일 자동 삭제(여러 번의 수정 시 마찬가지)
**도면이 여러 개일 경우 > 1.전체 수정 또는 2.일부 수정의 케이스가 있음으로 > 선택 해서 수정 가능 해야 함(여러 도면 중 N개의 도면 수정 시 > 수정 된 도면만 이전 파일 삭제 > 수정 된 도면만 발송 되게) >>최종적으로 최종 도면만 첨부파일 ‘도면’에 첨부 되게
